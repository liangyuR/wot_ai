# 常量配置重构

## 目标

将 `navigation/common/constants.py` 与 `navigation/common/overlay_config.py` 中的配置参数统一迁移到 `config.yaml`，通过加载配置提供默认值，减少魔法数字。

## 实施步骤

1. **扩展配置模板**  

- 在 `config.yaml.template` 中新增 sections：`constants`（检测、路径、控制等参数）与 `ui.overlay`（宽高、透明度等）。
- 为每个现有常量提供键值，并保留默认值与注释。

2. **实现配置读取逻辑**  

- 新建 `navigation/common/config_loader.py`（或在现有 loader 中扩展）从 YAML 中解析上述 section。  
- 提供数据类/访问接口（如 `NavigationConstants`, `OverlayConfig`）来封装读取的值，保留默认 fallback。

3. **更新 `constants.py` 和 `overlay_config.py`**  

- 修改为从配置对象中读取值，而不是硬编码。  
- 允许在未提供配置时退回默认值，维持向后兼容。

4. **注入配置到使用方**  

- 更新依赖这些常量的模块（如 overlay、导航逻辑）以接受新的配置对象或通过集中入口获取。  
- 确保 TaskManager / MainWindow 加载配置后传递到导航组件。

5. **验证**  

- 手动检查关键模块能读取配置值并正确运行。  
- 更新文档/日志，提示从配置加载的参数。
